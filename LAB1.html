Conceptos

URL: Es un acronimo de Uniforma Resources Locator y es una referencia como una direccion de un recurso de Internet[1].

MIME: (multipurpose Internet mail extensions) es un estandar para el formateo de diferentes tipos de archivos, tal como texto, graficos o audio, entonces estos pueden ser mandado por Internet y ser vistos o ejecutados por un navegador y expandir principalemnte las capacidades limitadas del correo electronico. Entonces lo que permite es describir el tipo de contenido del mensaje y el tipo de codigo usado con encabezados que le permite ser interpretado por el recpetor.[2]

XML: (Extensible Markup Language) formato universal para datos y documentos, que utiliza etiquetas para estrcuturar datos del documento[3]. Al igual que es un lenguaje entendido por los humanos y las computadoras

XHTML: Es HTML expresaco como XML, donde su objetivo es lograr una web semenatica.

HTML5: es usado para estrucutrar y presentar el contenido para la web,no obstante es la quinta version, donde navegadores como chrome, firefox, explorer y safari pueden saber la forma de mostrar determinada pagina, al igual que ya hace diferencia entre la estrucura, semantica, y js[5].

JS: Es un lenguaje de programacion que te permite realizar ciertas acciones en una pagina web, puesto que deja de mostrar solo informacion estatica[6].

PHP: Es un lenguaje que usa las etiqueta para entrar y salir del modo PHP. Donde su primer caracteristica es que el codigo es ejecutado en el servidor, esto genera HTML que es enviado al cliente[7].

JSP: Jva Server Pages, es una tecnologia que busca crear paginas web con programacion java, asi mismo con JSP podemos crear aplicaciones web que puedan ser ejecutadas en diversos servidores

Ajax: Asynchronous Javascript and XML, es una tecnica para el desarrollo de paginas web cuando se implementan aplicaciones interactivas, y en lo que nos ayua es que una pagina web que ya ha sido cargada solicite nueva informacion al servidor, con la ventaja que podemos actualizar solo la capa que hemos sugerido, sin necesidad de cargar toda la pagina.[8].

Otros conceptos

Internet vs web Internet por su parte es un sistema que hace una gran red de comunicacion, mientras que web es un sistema que funcina por medio del internet en donde se pueden transmitir diversos tipos de datos gracias a sus protocolos[9].

metodos HTTP GET, recuperan datos;HEAD, pide respuesta identica a la peticion Get;POST, se utiliza para enviar una entidad o un recurso especifico;PUT Y PATCH, se utilizan para amplicar modificaciones parciales a un recurso;DELETE, Borra un recurso especifico[10].

HTTP con código 200 Significa la respuesta estandar para peticiones correctar, "OK"

HTTP con código 404 Se utiliza cuando el servidor web no encuentra la pagina o recurso solicitado.

HTTP con código 500 Se utiliza cuando hay un error en el servidor

Atriubuto despreciado Significa que el atributo no es recomendable usarlo debido a que algunos navegadores puede ser que ya no lo soporten,un ejemplo es acronym, basefont, entre otros.

HTML4 VS HTML5 En HTML4 se encontraba todo aspecto combinado, que son estrucutra, estilos, entre otros. HTML5 lo que hace es una separacion en los aspectos anteriores.

componentes de estructura y estilo tiene una tabla THEAD, TBODY, TFOOT

Principales controles de una forma HTML5 nombre, valor, solo lectura, indice de la tabla
